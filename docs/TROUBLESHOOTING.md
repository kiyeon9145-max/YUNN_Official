# 자주 겪는 에러 해결법 (초보자용)

초보 개발자 분들이 YUNN MVP 프로젝트를 진행하면서 겪을 수 있는 흔한 문제들과 해결 팁을 모아두었습니다.

---

### Q1. "사진 업로드가 안 되고 에러가 납니다!"
**원인:** 브라우저의 로컬 스토리지(`localStorage`)는 보통 5MB 정도의 용량 제한이 있습니다. 고화질 사진(10MB 이상)을 그대로 넣으려고 하면 `QuotaExceededError`가 발생합니다.
**해결 방법:** 
1. 개발 중 테스트할 때는 용량이 작은 이미지(1MB 이하)를 사용하세요.
2. 기존에 꽉 찬 데이터를 지우려면: 크롬 브라우저에서 `F12(개발자 도구)` ➔ 상단 `Application` 탭 ➔ 좌측 `Local Storage` ➔ 항목 우클릭 후 `Clear`를 선택하고 새로고침하세요.

### Q2. "자바스크립트나 CSS 코드를 수정했는데 화면에 반영이 안 됩니다!"
**원인:** 브라우저가 인터넷 속도를 높이기 위해 이전 코드를 기억하고 있는 '캐시(Cache)' 현상 때문입니다.
**해결 방법:** 
강력 새로고침을 진행하세요.
- Windows/Linux: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

### Q3. "JS 파일에서 Uncaught TypeError: Cannot read properties of null 이 뜹니다!"
**원인:** `document.getElementById('아이디')` 로 찾으려는 HTML 태그가 해당 화면(html 파일)에 존재하지 않거나, 오타가 났기 때문입니다.
**해결 방법:**
1. 에러가 난 줄(Line 번호)을 확인합니다.
2. 그 줄에서 찾고 있는 ID가 HTML 파일에 정확히 똑같이 적혀 있는지 확인하세요. (대소문자 구별)

### Q4. "날짜 계산 로직이 자꾸 어긋납니다!"
**원인:** 자바스크립트의 `new Date()`는 사용자의 현재 기기 시간에 의존하며, 시간 단위(밀리초)까지 계산하므로 날짜 단위의 단순 비교가 까다롭습니다.
**해결 방법:** 
시간 단위까지 자르고 연/월/일만 비교하도록 `setHours(0,0,0,0)`을 사용하거나, YYYY-MM-DD 형식의 문자열로 만들어 비교해보세요.
