# YUNN MVP PRD 및 온보딩 가이드
*(입문 개발자 온보딩형 PM + Tech Lead 모드)*

안녕하세요! YUNN 프로젝트에 오신 것을 환영합니다. 이 문서는 코딩을 처음 접하거나 아직 익숙하지 않은 분들도 1~2주 안에 YUNN의 핵심 기능(MVP)을 이해하고 실제로 만들어 볼 수 있도록 작성되었습니다. 너무 어려운 개념은 빼고, 당장 화면에 보이고 동작하는 것 위주로 친절하게 설명해 드릴게요.

---

## 1) 프로젝트 개요

### 목표
인도 델리 지역의 2030 상위층 여성들이 겪는 여드름과 색소침착 문제를 한국 화장품(K-뷰티)과 맞춤형 스킨케어 루틴으로 해결하는 MVP 웹 서비스를 구축합니다.

### 타깃
- 인도 델리 거주 2030 상위층 여성
- 여드름/색소침착 고민이 있으며 K-뷰티에 관심이 있는 사용자

### 핵심 문제
단순히 화장품을 파는 것을 넘어, "어떻게 제대로 피부를 관리해야 하는가"를 알려주고 매일 실천하도록(강제성 부여) 돕는 것이 핵심입니다.

### 성공 기준(KPI 3개)
1. **설문/AI 진단 완료율:** 진단을 시작한 사람 중 끝까지 완료하는 비율 70% 이상
2. **루틴 알림 확인율:** 매일 발송되는 스킨케어 루틴 알림을 클릭/확인하는 비율 50% 이상
3. **재방문율:** 2주 뒤 비포/애프터 비교를 위해 다시 사이트에 접속하는 비율 30% 이상

## 2) 범위 정의

### 이번 MVP에 포함(In scope)
- 간단한 설문 및 사진 업로드 UI를 통한 (가상) 피부 진단 기능
- 진단 결과에 따른 화장품 추천 및 스킨케어 루틴 자동 설계 화면
- 브라우저 기반 알림(Web Push) 또는 시간에 맞춘 알림 모달(웹 페이지 접속 시)
- 2주 전/후 사진 비교(업로드 및 나란히 보기) UI

### 이번 MVP에서 제외(Out of scope)
- 실제 복잡한 머신러닝 AI 모델 연동 (우선은 하드코딩된 로직이나 간단한 조건문으로 대체)
- 실제 결제 시스템(PG) 연동 (결제 버튼 클릭 시 "준비 중입니다" 팝업)
- 백엔드 서버 DB (우선은 브라우저의 `localStorage`를 사용하여 데이터를 저장하고 불러옴)

## 3) 사용자 시나리오
> **초보자 설명:** 사용자가 앱을 켜서 끌 때까지 어떤 행동을 하는지 이야기 형식으로 정리한 것입니다.

### 스토리 1: 피부 진단받기
- **Given(상황):** 여드름이 고민인 아냐냐(Ananya)가 처음 사이트에 접속했다.
- **When(행동):** 내 피부 상태 설문(건성/지성 등)에 답하고 얼굴 사진을 1장 업로드한다.
- **Then(결과):** "아냐냐 님은 여드름성 지성 피부입니다"라는 결과와 추천 K-뷰티 제품 리스트가 나온다.

### 스토리 2: 스킨케어 루틴 확인 및 알림 설정
- **Given(상황):** 진단을 마친 아냐냐가 추천 제품을 확인했다.
- **When(행동):** '내 루틴 시작하기' 버튼을 누른다.
- **Then(결과):** 아침/저녁으로 해야 할 스킨케어 순서가 나타나고, 매일 아침 8시, 저녁 9시에 알림을 받기로 동의한다.

### 스토리 3: 2주 후 변화 확인
- **Given(상황):** 2주 동안 루틴을 따라한 아냐냐가 알림을 받고 접속했다.
- **When(행동):** 현재 얼굴 사진을 새로 업로드한다.
- **Then(결과):** 2주 전 사진과 현재 사진이 나란히 보이며, 피부가 얼마나 개선되었는지 응원 메시지가 나온다.

## 4) 기능 명세

### 기능 1. AI 진단 설문 및 사진 업로드
* **초보자 설명:** 사용자에게 질문을 던지고, 사진을 올려서 내 피부 상태를 입력받는 화면입니다.
* **왜 필요한가:** 맞춤형 화장품과 루틴을 제공하기 위한 첫 단추입니다.
* **입력값:** 설문 선택값(버튼 클릭), 이미지 파일 1장
* **출력값:** 진단 결과 데이터 (예: "여드름성 피부")
* **UI 요소:** 질문 텍스트, 선택 버튼, 사진 파일 첨부 버튼(`<input type="file">`), '결과 보기' 버튼
* **상태 변화:** 설문 진행도(Progress bar) 증가
* **예외 처리:** 사진을 안 올리고 넘어가려 하면 "사진을 업로드해주세요!" 경고창 띄움.
* **로그 포인트:** '결과 보기' 버튼 클릭 시 이벤트 기록

### 기능 2. 결과 기반 추천 및 루틴 설계
* **초보자 설명:** 진단 결과를 바탕으로 어떤 화장품을 써야 하는지, 아침/저녁에 어떻게 바르는지 보여줍니다.
* **왜 필요한가:** 사용자에게 직접적인 해결책(제품 + 관리법)을 제공합니다.
* **입력값:** 진단 완료 데이터
* **출력값:** 추천 제품 목록 이미지, 아침/저녁 스텝별 텍스트
* **UI 요소:** 제품 카드 리스트, 아침/저녁 탭 버튼, 체크리스트
* **상태 변화:** '루틴 시작' 누르면 브라우저 저장소(localStorage)에 오늘부터 시작함(Day 1)을 저장.
* **예외 처리:** 만약 진단 데이터가 없으면 처음 설문 화면으로 돌려보냄.

### 기능 3. 매일 알림 기능 (알림창)
* **초보자 설명:** 정해진 시간에 웹사이트 접속 시 "스킨케어 할 시간이에요!"라고 알려줍니다. (실제 푸시 대신 MVP용 웹 알림)
* **왜 필요한가:** 강제성과 습관을 만들어주기 위함입니다.
* **입력값:** 현재 시간, 사용자의 루틴 진행 일차
* **출력값:** 화면 중앙에 뜨는 알림 팝업(모달)
* **UI 요소:** "스텝 1: 클렌징 할 시간입니다" 모달 창과 '완료' 버튼
* **상태 변화:** '완료' 버튼 누르면 오늘 루틴 체크 완료 처리
* **예외 처리:** 이미 오늘 루틴을 완료했으면 다시 알림을 띄우지 않음.

### 기능 4. 2주 후 비포/애프터 비교
* **초보자 설명:** 첫날 찍은 사진과 14일 차에 찍은 사진을 옆에 나란히 두고 비교합니다.
* **왜 필요한가:** 제품과 루틴의 효과를 눈으로 보여주어 신뢰도를 높이고 재구매를 유도합니다.
* **입력값:** 새로운 사진 파일
* **출력값:** (과거 사진) | (현재 사진) 비교 화면
* **UI 요소:** 두 개의 이미지가 나란히 있는 레이아웃, 축하 애니메이션(꽃가루 등)
* **예외 처리:** 첫날 올린 사진 기록이 없으면, "이전 사진이 없습니다" 안내 텍스트 노출.

## 5) 화면/플로우

### 화면 목록
1. **홈 화면 (`index.html`)** : 메인 배너, 진단 시작하기 버튼
2. **진단 화면 (`quiz.html`)** : 설문 진행, 사진 업로드 
3. **결과 화면 (`result.html`)** : 추천 제품 및 루틴 확인, 루틴 시작 버튼
4. **마이 루틴 화면 (`routine.html`)** : 매일 접속해서 체크하는 출석부 및 체크리스트 화면, 2주차엔 비포/애프터 업로드 기능 활성화

### 화면 이동 흐름
`홈(index)` ➔ (버튼 클릭) ➔ `진단(quiz)` ➔ (설문/사진 완료) ➔ `결과(result)` ➔ (루틴 시작) ➔ `마이 루틴(routine)`

### 필수 컴포넌트 (공통)
- 상단 헤더 (로고, 뒤로가기)
- 하단 네비게이션 바 (홈, 쇼핑, 루틴, 마이페이지)

## 6) 데이터/저장 구조
> **초보자 설명:** 복잡한 서버 DB 대신, 브라우저에 임시로 정보를 저장하는 `localStorage`라는 공간을 사용합니다. 컴퓨터를 끄거나 브라우저 탭을 닫아도 정보가 남아있습니다.

**최소 데이터 모델 (키워드: `yunn_user`)**
| 필드명 | 타입 | 필수 여부 | 예시값 | 설명 |
|---|---|---|---|---|
| `skinType` | 문자열(String) | O | "oily" | 설문 결과 (지성 등) |
| `photoBefore` | 문자열(Base64) | O | "data:image/jpeg;base64..." | 첫날 업로드한 사진 데이터 |
| `photoAfter` | 문자열(Base64) | X | null | 14일 차에 업로드한 사진 |
| `routineStartDate`| 날짜(String) | O | "2026-04-30" | 루틴 시작 날짜 |
| `dailyCheck` | 배열(Array) | O | [true, false, true...] | 매일 스킨케어를 했는지 체크 |

> **개인정보 처리 주의점:** 사진 데이터는 민감 정보입니다. MVP에서는 서버로 전송하지 않고 사용자 본인의 기기(브라우저)에만 안전하게 저장합니다.

## 7) API 또는 이벤트 계약
> **초보자 설명:** 프론트엔드 HTML과 데이터를 다루는 JavaScript 간의 약속입니다. 서버 대신 JS 함수로 만듭니다.

**1. 진단 결과 저장 함수**
- **엔드포인트(함수명):** `saveQuizResult(data)`
- **요청 예시(JSON):** `{ "type": "acne", "image": "data:image..." }`
- **응답 예시:** 성공 시 `true` 반환
- **오류 메시지 규칙:** 이미지 용량이 너무 클 경우 "Error: 이미지가 너무 큽니다." 알림

**2. 오늘 루틴 완료 처리 함수**
- **엔드포인트(함수명):** `checkDailyRoutine(day)`
- **요청 예시(JSON):** `1` (1일차 완료)
- **오류 메시지 규칙:** "Error: 아직 루틴을 시작하지 않았습니다."

## 🎓 [초보자 학습 포인트 TOP 5]
1. **HTML 시맨틱 태그:** `div`, `section`, `button`, `input`이 언제 쓰이는지 감 잡기
2. **CSS Flexbox:** 항목들을 가로로 나란히 놓고 중앙 정렬하는 `display: flex`, `justify-content` 완벽 이해하기
3. **JavaScript DOM 조작:** JS로 HTML 요소를 찾고(`getElementById`), 버튼에 이벤트를 다는 법(`addEventListener`) 배우기
4. **로컬 스토리지(localStorage):** 브라우저에 임시로 데이터를 저장하고(`setItem`), 꺼내오는(`getItem`) 방법 익히기
5. **모바일 뷰포트:** 기기 크기에서 깨지지 않게 `@media` 쿼리와 `max-width` 사용하는 법 연습하기
